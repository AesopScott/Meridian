# Meridian UI Authority

`index.html` at the repository root is the actual Meridian user interface.

This is the UI Scott has been building and using. It owns the visible Meridian cockpit: the harness dock, Prime panel, user/session panel, Spark center, right-panel modes, model harness views, local bridge status, and backend-sourced Prime/Relay/Compass/Vulcan runtime logic surfaces.

Electron is a wrapper around this same UI. `npm start` must open `index.html` through `electron/main.js` and must not regenerate or substitute another HTML file before launch.

`bifrost/preview.html` is different. It is generated by `bifrost/preview.py` from backend sample/view-model data for deterministic rendering proofs and review. It is not the product UI entrypoint and should not be used to demo or operate Meridian unless a task explicitly asks for generated Bifrost preview proof output.

When changing UI behavior, start with:

- `index.html`
- `docs/ui-integration-checklist.md`
- `scripts/meridian-model-bridge.js`
- `electron/main.js` when desktop launch behavior changes
- `tests/test_bifrost_cockpit.py` and `tests/test_bifrost_preview.py` for guardrails

If these files disagree, this document is the tie-breaker: `index.html` is the actual Meridian UI.
