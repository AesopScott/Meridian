# Non-Orchestrator Surface Naming

The current phrase "non-orchestrator window" is accurate but not product-worthy.

This surface is still a prompt window. It is where Prime places artifacts, plan reviews, proof, cross-check findings, system messages, comparisons, gates, and other items Scott may need to inspect or respond to outside the main Prime conversation.

## Requirements

The name should communicate:

- review/gating
- proof/evidence
- promptability
- system findings
- not worker-session management
- not a passive log

Automatic cross-check findings should appear here.

## Candidate Names

### Review Console

Plain and honest.

Good for plans, cross-check, proof, and approval prompts.

### Gate Console

Strong for approval/rejection, but may sound too narrow.

### Review Bay

Cockpit/starship friendly. Suggests a place where items dock for inspection.

### Proof Console

Strong with Aegis, but too narrow if the surface also holds plans and system prompts.

### Triage Console

Good for findings and decisions, but can feel emergency-oriented.

### Inspection Console

Accurate for artifacts, proof, plans, and system findings. Slightly formal.

### Review Queue

Clear, but misses that it is a prompt window and interactive surface.

### Gate Queue

Clear for human-gated items, but too narrow for cross-check findings or system messages.

### Workbench

Promptable, practical, artifact-oriented. Less cockpit-specific.

### Dock

Nice metaphor for artifacts arriving, but maybe too abstract.

## Current Favorite

`Review Console`

It is honest, understandable, and broad enough:

```text
Orchestrator Queue: conversation with Prime.
Review Console: promptable review/gating surface for artifacts, proof, cross-check, system findings, and gates.
```
